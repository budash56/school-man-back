Current Project Scan (src/)

**src (root files)**
Feature: App bootstrap, module wiring, global validation, Swagger setup, TypeORM root options.
Classes: AppController, AppService, AppModule.
Methods: getHello, buildTypeOrmRootOptions, bootstrap.
Medium priority issues:
- TODO: Consider environment-specific env files (e.g., `.env.test`) via `ConfigModule.forRoot({ envFilePath: ... })` so tests can cleanly target a separate DB without manual env overrides.
Low priority issues:
- TODO: Confirm `migrationsRun: true` is intended in production; auto-running migrations on boot can be risky without explicit release control.

**src/auth**
Feature: Authentication (login/signup/me), JWT strategy, role guards, access checks.
Classes: AuthController, AuthService, JwtStrategy, JwtAuthGuard, RolesGuard, AccessService, AuthModule, DTOs.
Methods: login, signup, me, validateUser, resolveSignupRole, isTeacherOfCourse, classGroupIdsForTeacher, courseIdsForTeacher.
High priority issues:
- TODO: Access checks assume numeric IDs (`userId` is converted to `Number`), but `nationalId` is a string. If non-numeric IDs are used, teacher access checks can silently fail. Refactor AccessService and controllers to use string IDs consistently.
Medium priority issues:
- TODO: JwtStrategy reads `JWT_SECRET` directly from `process.env`. Consider using ConfigService for consistency with other configuration access patterns.

**src/attendance**
Feature: Attendance CRUD, roster sheet generation, write locks, teacher scoping.
Classes: Attendance, AttendanceController, AttendanceService, AttendanceRosterService, AttendanceRepository, DTOs.
Methods: create, findAll, findOne, update, remove, buildRoster.
Low priority issues:
- TODO: Legacy attendance path (slotId = null) is still supported. Consider formal deprecation plan if this path should be removed.

**src/grades**
Feature: Grades CRUD, letter mapping, write locks.
Classes: Grades, GradesController, GradesService, GradesRepository, DTOs.
Methods: create, findAll, findOne, update, remove.
High priority issues:
- TODO: `GradesService` throws a generic `Error` when term marks are missing. Replace with a typed HTTP exception (e.g., `BadRequestException`) so clients get a consistent 4xx response instead of a 500.

**src/reports**
Feature: Grade reports + certificates, print ID generation.
Classes: ReportsService, PrintIdService, GradesReportsController, CertificatesController, ReportsModule, DTOs.
Methods: getTermReport, getFinalReport, generateActiveStudentCertificate, nextId.
High priority issues:
- TODO: `PrintIdService` throws a generic `Error` if `print_generation_seq` is missing; consider returning a controlled 5xx/4xx with a clear remediation message or verifying sequence existence during startup/migration health checks.
Medium priority issues:
- TODO: Certificate endpoint returns `todos` (PDF body/signatures missing). Add real certificate rendering or integration with the PDF pipeline.
- TODO: Reports controller converts `nationalId` to number for access checks; this can break teacher access if IDs are not numeric. Align access checks to use string IDs consistently.

**src/notifications**
Feature: Notifications CRUD + absence-suggestion monitor.
Classes: NotificationsController, NotificationsService, NotificationsAbsenceMonitorService, NotificationsRepository, DTOs.
Methods: create, findAll, findOne, update, remove, resolve, runAbsence, evaluate.
Low priority issues:
- TODO: Debug logging in absence monitor is gated to `NODE_ENV=test`; confirm this is acceptable for CI noise or add a more explicit debug flag.

**src/timetable_generator**
Feature: Timetable preview/apply generator.
Classes: TimetableGeneratorController, TimetableGeneratorService, DTOs.
Methods: preview, apply.
Issues: none observed.
